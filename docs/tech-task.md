# Техническое задание: Десктоп-приложение Yappari (Интервальное повторение: Японский -> Русский)

## 1. Общая концепция

Минималистичное десктоп-приложение для заучивания японских слов методом интервального повторения.  
**Платформа:** Windows 11 (нативная оболочка Wails + WebView2).  
**Языки:** японский -> русский (только в одну сторону).  
**Пользователи:** однопользовательское, локальная БД SQLite.
**Название:** Yappari
**Dev-название:** yappari
**Статус:** MVP завершён, версия 0.1
**Ключевые особенности:**

- Алгоритм SM-2 с ручной оценкой (grade 0 / 3 / 4 / 5).
- Фуригана через ручной ввод + отображение `【кана】`.
- Автоматическая озвучка (Microsoft Edge TTS через edge-tts) при показе ответа.
- Два режима тренировки: интервальное повторение и свободный режим (с опциональным автовоспроизведением).
- Тёмная тема ( #000000 / #ffffff / #c7cdd8 / #ff0a14 ).
- TAB-навигация по интерфейсу.

## 2. Стек технологий и архитектура

| Слой | Технология | Комментарий |
| ---- | ---------- | ----------- |
| Desktop runtime | Wails v2+ (Go + WebView2) | Прямой IPC между Go и JS |
| Backend (бизнес-логика, БД, TTS) | Go | Чистый Go, драйвер SQLite без CGO (`modernc.org/sqlite`) |
| Database | SQLite (файл `database.db` в `AppData/Local/Yappari`) | Все данные локально |
| Frontend | Vue.js 3 (Vite) + TypeScript | Без серверного рендеринга |
| UI-компоненты | PrimeVue 4 | Диалоги, инпуты, прогресс-бары (кастомная тёмная тема через CSS) |
| Стилизация | Чистый CSS | Переопределяем стили PrimeVue под тёмную палитру |
| Шрифты | Inter + Noto Sans JP | Подключены локально из `frontend/public/fonts/` |
| Синтез речи | edge-tts (Microsoft Edge TTS) | Go вызывает edge-tts CLI как subprocess, возвращает MP3 (base64) -> фронтенд воспроизводит через `Audio` API |
| Иконки | canvas-confetti | Анимация конфетти при завершении тренировки |

**Примечание:** CGO не используется, сборка на Windows без GCC. Для синтеза речи требуется Python + `pip install edge-tts`.

## 3. База данных (SQLite)

### 3.1. Таблица `decks`

```sql
CREATE TABLE decks (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    name TEXT NOT NULL,
    created_at TEXT NOT NULL DEFAULT (datetime('now'))
);
```

### 3.2. Таблица `cards`

```sql
CREATE TABLE cards (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    deck_id INTEGER NOT NULL,
    kanji_text TEXT NOT NULL,          -- Полное написание слова (например, "食べる")
    furigana_text TEXT,                -- Полное чтение каной (например, "たべる"), может быть NULL
    translation TEXT NOT NULL,         -- Русский перевод
    -- Поля SM-2
    ease_factor REAL NOT NULL DEFAULT 2.5,
    interval INTEGER NOT NULL DEFAULT 0,   -- дней
    repetitions INTEGER NOT NULL DEFAULT 0,
    next_review TEXT NOT NULL,             -- ISO8601 ("2026-07-10T10:00:00Z") или NULL, если ещё не изучалась
    last_review TEXT,
    created_at TEXT NOT NULL DEFAULT (datetime('now')),
    updated_at TEXT NOT NULL DEFAULT (datetime('now')),
    FOREIGN KEY (deck_id) REFERENCES decks(id) ON DELETE CASCADE
);
```

**Правила валидации:**

- `kanji_text` обязательно.
- Если `furigana_text` == `kanji_text` или равно `NULL`, считаем, что слово не содержит кандзи (только кана). На фронтенде фуригана не отображается.
- `next_review` обновляется после каждого повторения в режиме интервального повторения. В свободном режиме не меняется.

## 4. Бизнес-логика (Go Backend) и IPC

### 4.1. Экспортируемые методы (IPC)

В Go-структуре, которая биндится к Wails, реализуем:

| Метод | Параметры | Возврат | Описание |
|-------|-----------|---------|----------|
| `GetDecks` | – | `[]Deck` | Список всех колод |
| `CreateDeck` | `name string` | `Deck` | Создание новой колоды |
| `UpdateDeck` | `id int, name string` | – | Переименование |
| `DeleteDeck` | `id int` | – | Удаление колоды и всех её карточек |
| `GetCardsByDeck` | `deckID int` | `[]Card` | Все карточки колоды (без полей SM-2 кроме `next_review`?) – для управления |
| `CreateCard` | `card CardInput` | `Card` | Добавление карточки |
| `UpdateCard` | `id int, card CardInput` | – | Редактирование карточки |
| `DeleteCard` | `id int` | – | Удаление |
| `GetTrainingCards` | `mode string` ("interval"/"normal"/"lazy"), `deckIDs []int` | `[]TrainingCard` | Возвращает карточки для текущей сессии |
| `SubmitReview` | `cardID int, grade int` (0,3,4,5) | `Card` (обновлённая карточка) | Применяет SM-2 только в режиме "interval" |
| `ResetCardProgress` | `cardID int` | – | Сброс ease_factor, interval, repetitions, next_review = now |
| `ResetDeckProgress` | `deckID int` | – | Сброс всех карточек колоды |
| `CheckVoicesAvailability` | – | `VoiceStatus` (ja:bool, ru:bool) | (Заглушка, voice-check заменён на edge-tts) |
| `SpeakText` | `text string, lang string` | `string` (base64 MP3) | Синтезирует речь через edge-tts, возвращает base64-аудио |
| `CheckEdgeTTSAvailability` | – | `{available: bool, message: string}` | Проверяет, установлен ли edge-tts в системе |

**Особенности:**

- `GetTrainingCards` для `"interval"` выбирает только карточки с `next_review <= now()` из указанных колод. Для `"normal"/"free"` - все карточки из указанных колод (без проверки даты).
- Карточки возвращаются в случайном порядке (перемешиваются на бэке).
- `SubmitReview` обновляет поля SM-2 и `next_review`, `last_review`, `updated_at`. Валидирует grade (0, 3, 4, 5).

### 4.2. Алгоритм SM-2 (реализация в Go)

Вход: текущие `ease_factor`, `interval`, `repetitions`, `grade` (0,3,4,5).  
Выход: новые значения.

```text
if grade >= 3:
    if repetitions == 0:
        interval = 1
    elif repetitions == 1:
        interval = 6
    else:
        interval = round(interval * ease_factor)
    repetitions += 1
else:
    repetitions = 0
    interval = 1

ease_factor = ease_factor + (0.1 - (5 - grade) * (0.08 + (5 - grade) * 0.02))
if ease_factor < 1.3: ease_factor = 1.3

next_review = today + interval days
```

**Сброс прогресса:** устанавливает `ease_factor=2.5`, `interval=0`, `repetitions=0`, `next_review = now()` (чтобы карточка была доступна сразу), `last_review` очищается.

## 5. Frontend: компоненты и страницы

Используем Vue Router с тремя вьюхами:

### 5.1. Главная страница (`/`)

- Список всех колод с чекбоксами (состояние сохраняется в `localStorage`).
- Две большие кнопки:
  - "Повторение" (режим `interval` - SM-2)
  - "Свободный режим" (режим `free` - без SM-2, с опцией автовоспроизведения)
- Кнопка "+ Новая колода" открывает модалку (PrimeVue Dialog) с полем имени и валидацией.
- Валидация: название не пустое, минимум 2 символа, анимация shake при ошибке.
- Кнопка ⚙️ на каждой колоде ведёт на страницу управления колодой.
- Логотип (yappari_logo.png) в шапке.

### 5.2. Управление колодой (`/deck/:id`)

- Заголовок с именем колоды (редактируемое поле InputText с сохранением по Enter).
- Список карточек (сетка карточек, не таблица): отображение через FuriganaText, фонемный клик для озвучки.
- Кнопка "Добавить карточку" -> модальное окно с формой.
- Кнопка "Сбросить прогресс всей колоды" (с подтверждением).
- Кнопка "Удалить" колоду (с подтверждением).

### 5.3. Форма карточки (модальное окно, встроено в DeckManage.vue)

Поля:

- Японское слово (обязательное, `InputText`)
- Чтение (кана, опциональное)
- Перевод (обязательное)

Валидация: слово и перевод не пустые, анимация shake при ошибке.  
Форма используется как для создания, так и для редактирования.

### 5.4. Тренировка (`/training`)

- Заголовок с названием режима ("Повторение" или "Свободный режим").
- Прогресс-бар (PrimeVue ProgressBar): `текущая_карточка / всего`.
- Карточка с анимацией переворота (CSS 3D flip):
  - Лицевая сторона: `kanji_text` (крупно).
  - Оборотная сторона: фуригана `【кана】` + перевод.
- Кнопка "Показать ответ" -> переворот карты + озвучка (JP -> 500ms пауза -> RU).
- В режиме `interval`: 4 кнопки оценки с эмодзи:
  - `😵‍💫` Повторить (grade 0)
  - `🥺` Трудно (grade 3)
  - `😊` Хорошо (grade 4)
  - `😜` Легко (grade 5)
- В режиме `free`: только кнопка "Далее" + кнопка "Авто-режим" (автоматический показ и озвучка с таймером).
- Клик по слову/переводу на обороте - повторная озвучка соответствующей части.
- По завершении - анимация конфетти (canvas-confetti) и кнопка "Вернуться на главную".

## 6. Фуригана: frontend-рендеринг

На входе: `kanji_text` (например, `食べる`), `furigana_text` (`たべる`).
Реализован Vue-компонент `FuriganaText.vue`.

**Текущая реализация (упрощённая):**

- Если `furigana_text` отсутствует, равно `null` или совпадает с `kanji_text` - показываем только `kanji_text`.
- Иначе показываем `kanji_text【furigana_text】`.
- Компонент принимает три пропса: `KanjiText`, `FuriganaText?`, `Language?` (ja/ru).
- Язык влияет только на цветовую анимацию при ховере (CSS gradient).

**Планируемое улучшение (future):** рендеринг `<ruby>` с посимвольной разбивкой кандзи и фуриганы.

## 7. Синтез речи (edge-tts)

Синтез речи реализован через **edge-tts** (Microsoft Edge TTS) - Python-утилиту, которая вызывает облачный TTS-сервис Microsoft. Качество речи на японском и русском языках значительно выше встроенных голосов Windows.

**Архитектура:**

1. Пользователь нажимает "Показать ответ" (или кликает по слову/переводу) -> фронтенд вызывает IPC-метод `SpeakText(text, lang)`.
2. Go-бэкенд запускает `edge-tts` как subprocess:
   - `edge-tts --voice ja-JP-NanamiNeural --text "..." --write-media /tmp/speak.mp3`
   - `edge-tts --voice ru-RU-SvetlanaNeural --text "..." --write-media /tmp/speak.mp3`
3. Go читает .mp3 файл, кодирует в base64, возвращает строку на фронтенд.
4. Фронтенд воспроизводит через `new Audio('data:audio/mp3;base64,...')`.

**Голоса:**

- Японский: `ja-JP-NanamiNeural` (женский, стандартный нейронный)
- Русский: `ru-RU-SvetlanaNeural` (женский, стандартный нейронный)

**Последовательность озвучки на тренировке:**

1. Произнести японское слово (`kanji_text`).
2. Дождаться окончания воспроизведения.
3. Пауза 500 мс.
4. Произнести русский перевод.
5. Дождаться окончания.

**Проверка доступности:**

- При старте приложение вызывает `CheckEdgeTTSAvailability()`, который проверяет наличие `edge-tts` в PATH.
- При отсутствии выводится сообщение с инструкцией по установке: `pip install edge-tts`.

**Зависимость:** Для работы требуется Python 3.x + пакет `edge-tts`. Устанавливается однократно.

## 8. Режимы тренировки

### 8.1. Интервальное повторение (mode = `"interval"`)

- Карточки загружаются через `GetTrainingCards("interval", selectedDeckIDs)` - только те, у кого `next_review <= now()`.
- Порядок случаен (перемешивание на бэке).
- Пользователь жмёт "Показать ответ" -> переворот карты + озвучка (JP -> 500ms -> RU).
- Затем нажимает одну из 4 кнопок (grade 0 / 3 / 4 / 5). Вызывается `SubmitReview`, применяется SM-2.
- Карточка уходит из стопки; после ответа на все показывается итог с конфетти.

### 8.2. Свободный режим (mode = `"free"`)

- Загружаются все карточки выбранных колод (`GetTrainingCards("normal", ...)`).
- Случайный порядок.
- После показа ответа только кнопка "Далее".
- **Авто-режим:** кнопка toggle включает автоматический показ:
  1. Показывается вопрос (2 сек).
  2. Автоматически показывается ответ + озвучка.
  3. После завершения озвучки + 3 сек паузы -> переход к следующей карточке.
  - Кнопка "Остановить" (она же "Пауза") возвращает ручное управление.
- Данные SM-2 не меняются.

## 10. Дизайн и стилизация

**Палитра (тёмная тема, строго):**

- Фон: `#000000`
- Основной текст, заголовки: `#ffffff`
- Второстепенный текст, обводки, неактивные элементы: `#c7cdd8`
- Акцент (кнопки, выделения, ссылки): `#ff0a14`

**Анимации:**

- `fade-in` для появления фуриганы и перевода при показе ответа (CSS transition opacity 0.2s).
- Анимированный `ProgressBar` (PrimeVue с кастомным градиентом `#ff0a14 -> #ff3b45`).
- CSS 3D flip для переворота карточки на тренировке.
- Shake-анимация при ошибке валидации форм.
- Confetti (canvas-confetti) при завершении тренировки.

**Шрифты:**

- Inter (основной, для русского и UI).
- Noto Sans JP (для японского текста).
- Подключены локально из `frontend/public/fonts/`.

**PrimeVue-компоненты (через tree-shaking):**

- Button, Checkbox (кастомный), Dialog, InputText, ProgressBar.
- DataTable не используется - вместо него кастомная сетка карточек.
- Тема: кастомная через CSS-переопределения (PrimeVue CSS-переменные + `!important`).
